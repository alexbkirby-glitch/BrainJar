AI coding tools start every session blank — no memory of your past bugs or the patterns your stack requires. Lodestone fixes this: a personal Stone that grows with your work, seeds what matters, and connects to any AI tool you use. Starts small. Gets smarter every session.

HOW IT WORKS

01 — START
Your seedbank, your knowledge
Lodestone starts with a small base of patterns about how to build seeds — then grows through your real sessions. Each seed is a wrong→correct antipattern with an exact symptom. The more you work, the more useful it becomes. No account required. Works fully offline.
02 — SCORE
Injection value scoring
Before a pattern enters your context, Lodestone asks: is this worth the space? Relevance to your current task, bug severity, and how full your context window already is all feed into the decision. Critical bugs stay in even under pressure. Cosmetic issues need a strong match to justify the cost.
03 — INJECT
Three ways to use it
MCP server works with Claude Desktop, Cursor, Windsurf, Continue, and any other MCP-compatible tool — no CLI required. Drop seeds into your Stone through the interface. Capture a fix directly from a session with one click. Lodestone connects to what you already use, with your permission.

SHARE ON STONEHUB
Your seeds, discoverable by everyone.
StoneHub indexes any public GitHub repo tagged with lodestone. There's no submission form, no PR, no approval. Tag your repo and it appears automatically.
1 — CREATE A PUBLIC REPO
Your Stone's seeds/ folder and lodestone-meta.json are all StoneHub needs. Push them to a public GitHub repo. 
git init my-lodestone
cd my-lodestone
cp -r ~/my-lodestone/seeds .
node scripts/build-index.mjs
git add . && git commit -m "init"
git remote add origin https://github.com/you/my-lodestone
git push
2 — ADD THE TOPIC TAG
On your GitHub repo page: About → Topics → add lodestone. StoneHub searches GitHub for this topic every time someone loads it. Your stone appears within minutes — no waiting for approval.

Rebuild and push whenever you add seeds. StoneHub always fetches the latest lodestone-meta.json from your main branch.

THE IDEA

        

          
AI coding assistants start every session blank. You re-explain your architecture, re-describe your bugs, re-establish your conventions — every time. The more complex your project, the more time you spend on context instead of code.

          

          
Lodestone extracts that knowledge once and makes it available to any AI session automatically. Your bug history, your working patterns, your project rules — loaded before you type a word.

          

          
The SeedBank extends this to stacks you haven't even configured yet. — curated wrong→correct patterns across — technologies, indexed by the exact symptoms developers google when something breaks.

        

      

      

        
ARCHITECTURE

        

          Three composable modules:


          stack-detector — infers language, framework, domain from files. Two-pass: heuristic (free, instant) then LLM (one call).


          skill-generator — converts any project into a structured SKILL.md. Bug history is always the first priority section.


          lodestone-inject — scores seeds via cosine similarity (MiniLM-L6, runs entirely in your browser). Injects the top-k seeds above a relevance threshold into your session context.
        

      

      

        
MODEL SUPPORT

        

          Works with any OpenAI-compatible endpoint.


          Recommended: Anthropic claude-sonnet-4 or claude-haiku-4

          Budget: OpenAI gpt-4o-mini

          Free/private: Ollama with llama3.2 or mistral


          Seed scoring uses MiniLM-L6 embeddings which run entirely in your browser via Transformers.js — no API key required for that step. The symptom reverse index requires no embeddings at all.

      How the estimates work


      Tokens spent building: This project was built across a single extended AI conversation. A typical multi-hour build session of this depth uses 3–6M tokens of context. The 4.2M figure is based on the total message count, average response length, and the number of large file outputs generated. It is not measured — there is no API that records conversation token usage after the fact.


      Tokens saved per session: Each Lodestone seed is ~120 tokens of context. Without it, establishing the same understanding would take roughly 2–3 turns of Q&A (~300 tokens each way per turn). Conservative estimate: chunk × 400 tokens saved. A typical session injects 6–10 chunks + 17 universal = ~23 chunks → ~9,200 tokens saved per session. This is a rough lower bound; complex sessions save more.


      Running total: sessions × 9,200. Sessions are estimated from npm installs + GitHub traffic, both of which undercount significantly — most users load seeds directly without going through npm.

Every Claude session

The Lodestone extension detects your stack, fetches the most relevant antipattern seeds, and injects them before you send your first message. Automatic. No copy-paste. No manual setup per session.

  


  

    
WHAT IT DOES

    

      

        
DETECT

        
Auto-identifies your stack

        
Scans the conversation every few seconds for language keywords, code block types, and stack names. The badge turns copper when context is ready — no interaction needed.

      

      

        
SCORE

        
Injects only what's relevant

        
Instead of dumping the whole seed, it scores each chunk against the current conversation and injects only the top 8 most relevant patterns plus the universal baseline. Keeps context lean.

      

      

        
GAUGE

        
Tracks your context window

        
Estimates token usage from the conversation. The badge ring fills as context grows — amber at 70%, red at 85%. When large, it surfaces a summary prompt and new-chat handoff so nothing is lost when you start fresh.

CONTEXT WINDOW FEATURES

    

      

        
⚡ SUMMARISE

        
Appears at 70% context

        
Injects a structured summary prompt — task state, key decisions, working code, next steps. Send it, and Claude writes a handoff document you can carry to a new session.

      

      

        
↗ NEW CHAT

        
One-click fresh session

        
After a summary is captured, "New chat with summary" opens a fresh claude.ai tab and auto-injects the summary plus the original Lodestone context — zero manual copy-paste.

      

      

        
💡 MODEL REC

        
Switch suggestion

        
When context is large, the panel reads the current model and suggests alternatives based on task type — Haiku for wrap-up, Sonnet for code, Opus for complex reasoning.
