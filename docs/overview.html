▸ LODESTONE — HOW IT WORKS

Your AI starts every session
already informed.

Lodestone is a personal seed library that grows through your real coding sessions. Each seed encodes one mistake — the wrong approach, the correct fix, and the exact symptom — so your AI never repeats it. Seeds are injected into each session automatically via MCP, or manually via Project Instructions. No accounts. No cloud sync. Offline by default.

THE CORE MODEL — HOW LODESTONE WORKS
01
Start small
Your seedbank starts with ~70 universal and stack-specific seeds about how to build good seeds. Nearly empty is correct — monoculture starts from adopting someone else's full library.
→
02
Capture fixes
When you solve a bug, call capture_fix from your MCP-connected AI. One call saves the wrong pattern, correct fix, and symptom as a new seed. Your library grows from what you actually encounter.
→
03
Score before inject
Before a seed enters your context, Lodestone scores it: relevance to your current task, severity, and context pressure. Critical patterns stay in even under pressure. Cosmetic ones need a strong match.
→
04
Inject via MCP
The MCP server connects to Claude Desktop, Cursor, Windsurf, Continue, and any MCP-compatible tool. Seeds are surfaced automatically when you hit a matching symptom — no copy-paste required.
→
05
Graft selectively
Browse Stonehub for stones from others working in your stack. Review and approve individual seeds before they enter your library. You control what grows in your seedbank.
THREE WAYS TO INJECT SEEDS
MCP SERVER
Automatic — Claude Desktop & Claude Code
The MCP server exposes lookup_symptom, capture_fix, get_seed, and more. Your AI calls these automatically. Seeds surface when relevant without you doing anything.
npx -y lodestone-mcp · or: claude mcp add lodestone
PROJECT INSTRUCTIONS
Paste-and-go — Claude.ai chat
Paste one block into your Claude Project's instructions. Claude fetches the public symptom index and stack seeds on-demand. No MCP server required — works entirely with Claude's web access.
alexbkirby-glitch.github.io/lodestone → Home → "USE WITH CLAUDE"
PREFLIGHT CLI
Before Claude Code sessions
Run lodestone-preflight.mjs before Claude Code starts. It scores your seedbank against current git context and injects the top seeds into CLAUDE.md automatically.
alias claude='node ~/tools/lodestone-preflight.mjs && claude'
KEY FILES IN YOUR SEEDBANK
CONTEXT FILE — ALWAYS IN CLAUDE PROJECTS
lodestone-nano.md
~1,600 tokens. Critical seeds + full symptom index. Upload to any Claude Project or paste into CLAUDE.md. The front door to your seedbank for any AI session.
LOCAL · ALWAYS IN CONTEXT
EXTENDED CONTEXT — FOR TOKEN-HEAVY STACKS
lodestone-critical.md
Every critical and high-severity seed in full. Use this for stacks where you have dense seed coverage. Pairs with lodestone-nano.md or replaces it for critical sessions.
LOCAL · UPLOAD TO PROJECT
SEED DATA — FULL LIBRARY
seeds/{stack}.json
Complete seed data per stack including relationships, blast radius, confidence scores, and provenance. Used by the MCP server for BM25 + graph-expanded lookup. Also fetchable from alexbkirby-glitch.github.io/lodestone/seeds/{stack}.json.
LOCAL + PUBLIC MIRROR
SYMPTOM INDEX — FOR FAST LOOKUP
api/symptom-index.json
Reverse index mapping error message tokens to seed IDs. O(1) lookup without embeddings. The MCP server uses this for initial candidate selection before BM25 re-scoring.
PUBLIC · alexbkirby-glitch.github.io/lodestone/api/symptom-index.json

PRIVACY — WHAT GOES WHERE
Your seedbank is local by default. Seeds live in your filesystem under seeds/. The MCP server reads them at query time. Nothing is uploaded anywhere unless you explicitly publish a stone to Stonehub.

The Skill Generator runs entirely in your browser. File contents are sent only to your chosen model endpoint (Anthropic, OpenAI, or a local Ollama instance) for analysis — not to any Lodestone server, because there is no Lodestone server. The site is hosted on GitHub Pages.

Stonehub publishing is explicitly opt-in. When you publish a stone, its seeds become publicly readable at your Stonehub profile URL. Seeds are patterns about code mistakes, not about your codebase itself — but review before publishing if you're cautious.
